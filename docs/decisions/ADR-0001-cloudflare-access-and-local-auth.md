# ADR-0001: Cloudflare Access とローカル認証を併用する

## ステータス

採用

## 背景

自宅 k3s クラスター上の Web アプリケーションを、外部ポートを開放せずにインターネットへ公開したいです。

通常時は Cloudflare Access によるゼロトラスト認証を利用したい一方で、LAN 内アクセスや Cloudflare 側の障害時にもアプリを利用できるようにしたいです。

## 決定

Cloudflare Access とアプリ独自のローカル認証を併用します。

- 外部公開経路では Cloudflare Access を最優先の認証手段とします。
- LAN 内アクセスやフォールバック用途ではローカル JWT 認証を利用します。
- 認証方式とアプリ内認可は分離し、権限は `role` で判断します。
- Cloudflare Access 経由で未登録ユーザーがアクセスした場合は、条件を満たす場合のみ JIT プロビジョニングします。

## 結果

メリット:

- 外部公開時の安全性を Cloudflare Access に任せられる。
- LAN 内でもアプリを通常利用できる。
- Cloudflare 側の障害時にもローカル認証で利用できる余地を残せる。

注意点:

- Cloudflare Access ヘッダーを信頼する境界を明確にする必要がある。
- FastAPI に直接到達できる経路を作らない必要がある。
- 可能であれば `Cf-Access-Jwt-Assertion` の署名検証を行う。
- JIT プロビジョニングの条件と初期ロールを厳格にする必要がある。
