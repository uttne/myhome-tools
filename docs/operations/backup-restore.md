# バックアップ・復旧

## 方針

バックアップ・復旧は既存 PostgreSQL の運用で扱います。

本アプリ側では、復旧時に必要な情報として DB 名、マイグレーション履歴、アプリバージョンの対応関係を把握できるようにします。

## アプリ側で確認する項目

- 専用 DB が既存 PostgreSQL のバックアップ対象に含まれること。
- 復旧時に Alembic の `alembic_version` テーブルも復元されること。
- 復旧後、アプリのバージョンと DB スキーマのバージョンが一致すること。

## 復旧時に必要な情報

TBD:

- 専用 DB 名
- 本番アプリのバージョン
- 適用済み Alembic revision
- 復旧対象日時
- 復旧確認手順

## 復旧確認

初期段階では、復旧手順は既存 PostgreSQL 側の手順に従います。

アプリ側では、復旧後に次を確認します。

- FastAPI が DB に接続できること。
- `/readyz` が成功すること。
- `/api/me` が認証状態に応じて期待通り応答すること。
- 主要データが参照できること。
