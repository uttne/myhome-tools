## 概要

認証後の業務画面共通レイアウトです。ヘッダーとコンテンツエリアに分けたシンプルな構成と、ハンバーガーメニューによるナビゲーションを提供します。

フェーズ 0 の認証専用 UI（`/` のログイン表示）を本レイアウトに置き換え、ログイン後は常にシェル内で画面遷移します。

## レイアウト構成

```text
┌─────────────────────────────────────┐
│ Header                              │
│  [≡]  myhome-tools    (ユーザー)    │
├─────────────────────────────────────┤
│                                     │
│ Content Area                        │
│  （各画面の内容）                     │
│                                     │
└─────────────────────────────────────┘

[≡] タップ → 左からドロワー（ハンバーガーメニュー）
```

### Header

| 要素 | 説明 |
| --- | --- |
| ハンバーガーボタン | ナビゲーションドロワーを開閉 |
| アプリ名 / ロゴ | `myhome-tools` |
| ユーザー表示 | ログインユーザー名または email（簡易） |

### Content Area

- 各 Route の `Outlet` を表示
- 画面ごとにパディング・最大幅を統一（例: `max-width: 960px`）

### ナビゲーションドロワー

| メニュー項目 | パス | フェーズ |
| --- | --- | --- |
| Home | `/home` | 1 |
| 買い物リスト | `/shopping` | 1 |
| ログアウト | `/logout` | 0（既存） |

将来のサービスはここに追加します。

## 画面

| 画面 | パス | 説明 |
| --- | --- | --- |
| AppShell | `/` 配下 | 認証済みユーザーの親レイアウト |

### ルーティング方針

```text
/                     未認証 → ログイン / 認証済み → /home へリダイレクト
/home                 Home（AppShell 内）
/shopping             買い物リスト（AppShell 内）
/shopping/...         買い物リスト子画面
/logout               ログアウト（AppShell 外でも可）
```

### 認証ガード

- AppShell 配下は認証必須
- 未認証アクセス時は `/` のログイン画面へ

## 技術方針

- React Router の nested routes + `Outlet`
- ドロワーは初期は CSS + state で実装（UI ライブラリは導入しない）
- スタイルは `styles.css` を拡張
