# グループ

| 項目 | 内容 |
| --- | --- |
| 機能 ID | groups |
| 状態 | 未着手 |
| フェーズ | 1（最小）、2（招待） |

## 概要

業務データのスコープ単位です。「家族」はグループの一種として運用します。

- 各ユーザーは **個人グループ** を1つ持つ（自動作成）
- ユーザーは **共有グループ** を作成し、メンバーを招待して家族単位でデータを共有する（招待はフェーズ2）
- 買い物リストなどのデータは `group_id` に紐づく

## コンセプト

| 概念 | 説明 |
| --- | --- |
| グループ | データ共有の単位。個人用と共有用がある |
| 個人グループ | ユーザー登録時に自動作成。1ユーザー1つ |
| 共有グループ | 管理者またはユーザーが手動作成。家族運用の主体 |
| メンバーシップ | ユーザーとグループの所属関係 |

## フェーズ別スコープ

### フェーズ 1

| 項目 | 内容 |
| --- | --- |
| 個人グループ | ユーザー作成（JIT / ローカル登録）時に自動作成 |
| 共有グループ | 管理者が手動作成。メンバー追加も手動（CLI または将来の admin API） |
| 招待 API | **なし**（フェーズ2） |
| データスコープ | 買い物リストは `group_id` 単位 |

### フェーズ 2 以降

- 共有グループの自由作成（全ユーザー）
- 招待リンク / 参加コード
- グループ切替 UI（現在操作中のグループ）

## API（フェーズ 1）

ベースパス: `/api/v1/groups`

| メソッド | パス | 概要 | 認可 |
| --- | --- | --- | --- |
| `GET` | `/api/v1/groups` | 自分が所属するグループ一覧 | user, admin |
| `GET` | `/api/v1/groups/{id}` | グループ詳細 | メンバー |
| `POST` | `/api/v1/groups` | 共有グループ作成 | admin（フェーズ1） |
| `POST` | `/api/v1/groups/{id}/members` | メンバー追加 | admin（フェーズ1） |

買い物リスト API は `group_id` をクエリまたはリクエストボディで受け取ります。未指定時のデフォルトグループは **未決**（初期は共有グループがあればそれを優先、なければ個人グループ）。

## データモデル

### groups

| カラム | 型 | 説明 |
| --- | --- | --- |
| `id` | UUID | PK |
| `name` | string | グループ名 |
| `kind` | enum | `personal`, `shared` |
| `created_by` | UUID | FK → users |
| `created_at` | timestamptz | |
| `updated_at` | timestamptz | |

制約: `kind = personal` のグループはユーザーあたり1つ（`group_memberships` 経由で保証）。

### group_memberships

| カラム | 型 | 説明 |
| --- | --- | --- |
| `id` | UUID | PK |
| `group_id` | UUID | FK → groups |
| `user_id` | UUID | FK → users |
| `joined_at` | timestamptz | |

制約: `(group_id, user_id)` UNIQUE。

### ER（概略）

```text
users
  └── group_memberships ──► groups
                                └── shopping_lists（他機能も group_id で紐づく）
```
