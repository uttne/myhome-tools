# 画面設計

## 概要

React + Vite + React Router で構成する SPA の画面一覧と遷移を整理します。

実装: `frontend/src/App.tsx`

## 画面一覧

| 画面 | パス | 概要 | 状態 |
| --- | --- | --- | --- |
| ホーム | `/` | 認証状態に応じてログイン画面またはログイン済み画面を表示 | 実装済み |
| ログアウト | `/logout` | ログアウト API を呼び出し、必要なら外部 URL へリダイレクト | 実装済み |
| メイン（業務） | TBD | ログイン後の業務機能画面 | 未実装 |
| 管理 | TBD | ユーザー管理や設定 | 未実装 |
| パスワード設定 | TBD | ローカルパスワードの設定・変更 | 未実装 |

## 初期ロード時の認証判定

アプリ初期ロード時（`/`）に FastAPI の `/api/me` を呼び出し、認証状態を判定します。

| `/api/me` の結果 | フロントエンドの挙動 |
| --- | --- |
| `200 OK` | ログイン済み画面を描画（email / role / provider を表示） |
| `401 Unauthorized` | ローカルログインフォームを表示 |
| その他のエラー | エラーメッセージを表示 |

Cloudflare Access 経由では Access 認証後に `/api/me` が成功するため、アプリ上ではシームレスにログイン済み状態になります。

## 画面遷移

```text
/ (HomePage)
├── 未認証 → ローカルログインフォーム
│   └── ログイン成功 → ログイン済み画面（同一ページ内で切替）
└── 認証済み → ログイン済み画面
    └── ログアウトボタン → /logout

/logout (LogoutPage)
└── POST /api/auth/logout → logout_url または / へリダイレクト
```

## ホーム画面（`/`）

### 未認証時

| 要素 | 内容 |
| --- | --- |
| タイトル | ローカルログイン |
| 説明 | Cloudflare Access を通らない場合はローカルアカウントでログイン |
| 入力 | メールアドレス、パスワード |
| 操作 | ログインボタン |
| エラー | 認証失敗時にメッセージ表示 |

### 認証済み時

| 要素 | 内容 |
| --- | --- |
| タイトル | ログイン済み |
| 表示 | Email、Role、Provider |
| 操作 | ログアウトボタン → `/logout` |

## ログアウト画面（`/logout`）

| 要素 | 内容 |
| --- | --- |
| タイトル | ログアウト中 |
| 動作 | マウント時に `/api/auth/logout` を POST |
| 成功 | `logout_url` があれば外部へ、なければ `/` へリダイレクト |
| エラー | 失敗メッセージを表示 |
