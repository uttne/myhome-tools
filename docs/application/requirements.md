# アプリケーション要件

## 概要

`myhome-tools` は、自宅 k3s 上で動作するホームラボ向け Web アプリケーションです。

家族など少人数が、Cloudflare Access 保護下のインターネット経路および LAN 内から利用できるツールを提供します。

アプリケーション設計の全体像は `design.md` を参照してください。

## 想定利用者

| 利用者 | 説明 |
| --- | --- |
| 管理者 | 初期設定、ユーザー管理、全機能の利用 |
| 家族ユーザー | 通常機能の利用 |

## 機能要件

### 実装済み

| ID | 機能 | 説明 |
| --- | --- | --- |
| FR-AUTH-001 | 認証状態確認 | `/api/me` で現在ユーザーを取得 |
| FR-AUTH-002 | ローカルログイン | メール + パスワードでログイン |
| FR-AUTH-003 | ログアウト | Cookie 削除、外部ログアウト URL 対応 |
| FR-AUTH-004 | パスワード設定 | ローカルパスワードの初回設定・変更 |
| FR-AUTH-005 | Cloudflare JIT | 許可メールの初回アクセス時にユーザー作成 |
| FR-AUTH-006 | 初期管理者作成 | CLI による管理者 seed |

### 未実装（認証・管理）

| ID | 機能 | 説明 | 優先度 |
| --- | --- | --- | --- |
| FR-ADMIN-001 | ユーザー一覧 | 管理者がユーザーを確認 | 中 |
| FR-ADMIN-002 | ユーザー無効化 | アカウントの有効/無効切替 | 中 |
| FR-ADMIN-003 | パスワードリセット | リセットトークン発行 | 中 |
| FR-AUTH-007 | JWT 検証 | Cloudflare Access JWT の署名検証 | 高 |

### 未決（業務機能）

業務機能の要件は未決です。`design.md` の「機能モジュール」で確定後、ここに FR-xxx 形式で追記します。

## 非機能要件

| ID | カテゴリ | 要件 |
| --- | --- | --- |
| NFR-001 | セキュリティ | 外部公開は Cloudflare Access 経由。API 直アクセス経路を作らない |
| NFR-002 | セキュリティ | JWT 署名鍵、DB 接続情報は Secret 管理 |
| NFR-003 | 可用性 | 単一 replica から開始。障害時はローカル認証で LAN 利用可能 |
| NFR-004 | 保守性 | Alembic マイグレーション、Helm Chart によるデプロイ |
| NFR-005 | バックアップ | 既存 PostgreSQL バックアップに専用 DB を含める |
| NFR-006 | 開発 | Docker Compose を標準開発環境とする |

## スコープ外

- マルチテナント提供
- 公開 SaaS 運用
- PostgreSQL インスタンス自体の運用
- k3s クラスター管理
